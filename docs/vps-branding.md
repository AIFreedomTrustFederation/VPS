# AIFT VPS branding guide

The public product brand should stay clear and technical:

```text
AIFT VPS
AI Freedom Trust VPS
Decentralized VPS Network
Mobile and Desktop VPS Nodes
```

## Primary brand

Use:

```text
AIFT VPS
```

AIFT VPS means a decentralized virtual private server network where phones, laptops, desktops, mini PCs, and servers can become provider nodes.

## Product sentence

Use this sentence when explaining the product:

```text
AIFT VPS is a decentralized VPS network that turns phones, laptops, desktops, and servers into provider nodes for cloud apps, edge compute, previews, builds, and verified hosting.
```

## Technical positioning

AIFT VPS should be described as:

- VPS provider network
- decentralized VPS platform
- cross-platform node network
- mobile and desktop compute network
- cloud app foundry on top of VPS nodes
- edge compute swarm backed by verified servers

## Avoid losing the VPS language

Do not brand the project only as:

- cloud app foundry
- mobile dashboard
- phone server
- edge app
- compute swarm

Those are features and layers. The technical anchor is VPS.

## Layered naming

Use this hierarchy:

```text
AIFT VPS
  -> AIFT VPS Dashboard
  -> AIFT VPS Node App
  -> AIFT VPS Provider Nodes
  -> AIFT VPS App Foundry
  -> AIFT VPS Edge Compute
```

## User-facing explanation

For technical users:

```text
AIFT VPS is building a decentralized VPS provider network where approved devices can contribute compute capacity and where every workload discloses where it runs.
```

For non-technical users:

```text
AIFT VPS lets people launch apps from a phone while using a transparent network of trusted devices and servers behind the scenes.
```

## Node branding

Use these labels:

```text
AIFT VPS Node
AIFT VPS Phone Node
AIFT VPS Desktop Node
AIFT VPS Server Node
AIFT VPS Verified Node
AIFT VPS Experimental Node
```

## App branding

The cross-platform app should be called:

```text
AIFT VPS Node App
```

The dashboard should be called:

```text
AIFT VPS Dashboard
```

The repository may still include app foundry language, but it should be presented as a feature of AIFT VPS.

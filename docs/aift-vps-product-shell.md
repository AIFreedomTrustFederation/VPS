# AIFT VPS product shell

AIFT VPS should feel familiar to technical users who know modern app platforms and VPS dashboards, while staying clearly branded as a decentralized VPS network.

## Product promise

```text
Create a project.
Connect source.
Build a preview.
Publish a release.
Review logs.
See node placement.
See disclosure.
```

## AIFT VPS difference

AIFT VPS is disclosure-first.

Every project should show:

- source repository
- template or framework
- build status
- preview status
- release status
- node placement
- node class
- runtime type
- logs
- disclosure record

## Core flow

```text
1. Create project
2. Pick template
3. Connect source
4. Configure build settings
5. Select workload level
6. Select node policy
7. Build preview
8. Inspect logs
9. Publish release
10. Review disclosure
```

## Workload levels

```text
Level 1: static files and public previews
Level 2: frontend build output
Level 3: dynamic preview process
Level 4: approved background job
Level 5: stateful service, verified nodes only
```

## First shell milestone

The first shell must clearly model:

- project creation
- source connection
- build preview
- release history
- node placement
- logs
- disclosure

This creates a familiar technical workflow while preserving the AIFT VPS network model.

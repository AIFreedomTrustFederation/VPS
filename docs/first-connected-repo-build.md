# First connected repository build

The first real repository to build through AIFT VPS is:

```text
AIFreedomTrustFederation/capital-city-provisions
```

This repository should not be copied as a meat, freezer, delivery, or giveaway brand inside AIFT VPS.

It should be used as a real connected source repository whose structure can be rebranded, filtered, and adapted into the AIFT VPS decentralized app-building system.

## Core rule

Build from the real repository, but do not preserve its original business meaning unless it applies to the AIFT VPS network.

AIFT VPS should extract reusable software patterns and reword them into VPS, decentralized compute, no-code app, provider-node, workload, region, and operator language.

## What to keep

Keep reusable architecture:

- Next.js app structure
- public page routing
- landing page gateway pattern
- multi-step intake pattern
- route or region checking pattern
- role-based assistant pattern
- operations lifecycle records
- owner/operator reporting pattern
- worker/provider workspace pattern
- open-source license audit
- mobile action policy
- verification command pattern

## What to discard or reword

Discard or reword original business-specific content:

- meat sales language
- freezer box language
- cheesecake offer
- giveaway-specific claims
- delivery promises
- customer order records
- private customer data
- original public brand copy
